java-多线程juc线程池

发表于 2017-12-23 | 更新于: 2017-12-24 | 分类于 java |

线程池的好处

  • 降低资源消耗，通过重复利用已创建的线程降低线程创建和销毁造成的损耗
  • 提高相应速度，当线程到达时候任务可以不需要等待线程创建就能立即执行
  • 提高线程的可管理性，统一分配、调优、监控

线程池实现原理

当向线程池提交一个任务，线程池的处理方式如下图所示

  • 如果线程池中的线程小于corePoolSize时就会创建新线程直接执行任务
  • 如果线程池中的线程大于corePoolSize就会暂时把任务存放在工作队列workQueue中等待执行
  • 如果工作队列workQueue也满时，当线程数小于线程池数maximumPoolSize时就会创建新线程来处理，而线程数大于最大线程池数maximumPoolSize时就会执行饱和策略。

参数说明：

  • corePoolSize，核心池的大小。 当有任务来之后，就会创建一个线程去执行任务，当线程池中的线程数目达到corePoolSize后，就会把到达的任务放到缓存队列当中。如果调用了线程池的prestartAllCoreThreads()方法，线程池会提前创建并启动所有基本线程。
  • maximumPoolSize(线程池最大数量),线程池允许创建的最大线程数。如果队列满了且当前线程数<最大线程，继续创建新线程。如果是无界队列，这个参数就无意义了，如果是有界队列，队列满了且线程数达到了最大线程数，在提交新任务将采取拒绝策略。
  • keepAliveTime,线程空闲后，保持存活的时间。任务很多，且任务较短，可以把这个时间调大。
  • workQueue(任务队列)，一个阻塞队列，用来存储等待执行的任务。
    ArrayBlockingQueue，一个由数组支持的有界阻塞队列。此队列按FIFO（先进先出）原则对元素进行排序
    LinkedBlockingQueue，是一个单向链表实现的阻塞队列，先进先出的顺序。支持多线程并发操作,比数组实现的ArrayBlockingQueue的有界，LinkedBlockingQueue可认为是无界队列。吞度量通常高于ArrayBlockingQueue。
    SynchronousQueue,一个没有数据缓冲的BlockingQueue(队列只能存储一个元素)，生产者线程对其的插入操作put必须等待消费者的移除操作take，反过来也一样，消费者移除数据操作必须等待生产者的插入
    PriorityBlockingQueue,是无界的阻塞队列。
  • ThreadFactory，用于设置创建线程的工厂
  • defaultHandler，饱和策略，当队列满了且maximumPoolSize已经达到最大，如果继续提交任务，线程池提供4种策略来处理
    abortPolicy:直接抛出异常，默认策略；
    CallerRunsPolicy:只用调用者所在的线程来运行任务。
    DiscardOldestPolicy:丢弃队列里最近的一个任务，并执行当前任务
    DiscardPolicy:不处理，丢弃掉。

向线程池提交任务

有两种方法，分别是execute()和submit()方法。

exeute和submit的区别？
execute用于提交不需要返回值的任务。无法判断任务是否执行成功。

submit用于提交需要返回值任务。线程池会返回一个future类型的对象。

线程池的关闭

调用shutdown()和shutdownNow()方法来关闭线程池，原理是遍历线程池中的工作线程，然后逐个调用线程的interrupt方法来中断线程，所以无法响应中断的任务可能永远无法终止了。

shutdown，只将线程池的状态设置成shutdown状态，中断所有没有正在执行的任务线程
shutdowNow，将线程池的状态改为stop状态，尝试停止所有的正在执行或者暂停的任务线程。

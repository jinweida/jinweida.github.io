区块链技术-什么是区块链？

发表于 2018-01-14 | 更新于: 2018-03-08 | 分类于 区块链 |

什么是区块链？

  • 区块链是一种分散式资料库
  • 最初广泛应用于比特币
  • 区块链维护一份连续不断的交易记录
  • 多笔交易构成一个区块(Block)
  • 每个区块都包含上一个区块的Hash值
  • 链接一起，区块与区块之间形成链条(Chain)
  • BC就是有多个区块形成的链条

区块链运作过程

  • 产生区块的过程是一个复杂的密码学运算(Cryptography)
  • 复杂的密码学运算可有效杜绝交易记录被篡改
  • 每个区块在成功产生后是无法修改的
  • 产生区块的过程称为挖矿(Mining)
  • 负责产生区块的计算机又称矿工(Miner)
  • 每一笔交易都可以透过链条找出可靠的历史记录
  • 由于是分散的资料库，所以具有去中心化的特征
  • 去中心和意味着资料会散布在多个节点
  • 意味着没人可以说出自己的区块才是合法有效的区块
  • 多数人验证过的区块才是合法有效的
  • 超过50%才是有效
  • 每个节点必须存储所有区块
  • 每个节点都可以协助验证区块的有效性
  • 任何一个资料被篡改都将破坏区块的完整性

区块链去中心化

  • 多个节点存储账本信息，构成一个网络
  • 容错力
  • 防勾结串通
  • 抗攻击力

链的分类

根据参与者不同可以分为公开链，联盟链，私有链

公开链

典型的是比特币区块链，信息完全公开。

私有链

集中管理者进行限制，只有内部少数人可以用，信息不公开

联盟链

介于上面两种之间，由若干组织一起维护一条区块链，使用者必须经过授权使用，典型的有hyperledger fabric。

锁机制

  • 悲观锁：假定会发生并发冲突，屏蔽一切可能违反数据完整性的操作。悲观锁的实现，往往依靠底层提供的锁机制；悲观锁会导致其它所有需要锁的线程挂起，等待持有锁的线程释放锁。
  • 乐观锁：假设不会发生并发冲突，每次不加锁而是假设没有冲突而去完成某项操作，只在提交操作时检查是否违反数据完整性。如果因为冲突失败就重试，直到成功为止。乐观锁大多是基于数据版本记录机制实现。为数据增加一个版本标识，比如在基于数据库表的版本解决方案中，一般是通过为数据库表增加一个 “version” 字段来实现。读取出数据时，将此版本号一同读出，之后更新时，对此版本号加一。此时，将提交数据的版本数据与数据库表对应记录的当前版本信息进行比对，如果提交的数据版本号大于数据库表当前版本号，则予以更新，否则认为是过期数据。

java-多线程sleep和wait区别

发表于 2018-01-14 | 更新于: 2018-01-14 | 分类于 java |

java多线程中，sleep是让线程休眠，经过指定时间后会继续执行，wait是等待，需要唤醒才能继续执行。

使用场景

sleep是Thread线程类的方法，而wait是Object顶级类的方法。
Thread.sleep()可以用在任何地方，而wait只能在同步方法或者同步块中使用。

CPU资源释放

sleep,wait 调用后都会暂停当前线程并让出cpu执行时间。不同的是sleep不会释放当前持有的对象的锁资源，到时间会自动执行。而wait会放弃所有锁资源，直到其他线程调用此对象的java.lang.Object.notify（）方法或java.lang.Object.notifyAll（）方法，或指定的时间已经过去。

异常捕捉

sleep需要捕获异常或抛出异常，而wait不需要捕获异常。

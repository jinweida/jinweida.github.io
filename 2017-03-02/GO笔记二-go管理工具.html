GO笔记二:go管理工具

发表于 2017-03-02 | 更新于: 2017-03-02 | 分类于 golang |

golang开发工具概览

go build

启动编译，生成可执行文件。使用go help build查看。
| 参数 | 说明 |
|—–|—–|
| -n | 查看但不执行编译。 |
| -x | 查看并执行编译命令。 |
| -a | 强制重新编译所有依赖包。 |
| -v | 查看被编译的包名，包括依赖包。 |
| -p n | 并行编译所使用CPU core 数量。默认全部。 |
| -o | 输出 件名。|
| -v | 查看被编译的包名，包括依赖包。|
| -gcflags | 传递给 5g/6g/8g 编译器的参数。|
| -ldflags | 传递给 5l/6l/8l 链接器的参数。|
| -work | 查看编译临时目录。|
| -race | 允许数据竞争检测 (仅 持 amd64)。|
下面三个执行的目录是同一个目录都是当前目录。

go clean

可以清除编译生成的文件。

使用go help clean。
| 参数 | 说明 |
|—–|—–|
| -n | 查看但不执行清理命令。 |
| -x | 查看并执行清理命令。 |
| -i | 删除 bin、pkg 目录下的二进制文件。 |
| -r | 清理所有依赖包临时文件。 |

go run

此命令是先编译在执行可以执行文件，go build只编译可执行文件，需要手动运行可执行文件

go fmt

使用 go fmt hello.go 格式化后

go env

查看当前go环境信息

go get

下载并安装扩展包。默认保存到 GOPATH 指定的第一个工作空间。
| 参数 | 说明 |
|—–|—–|
| -d | 仅下载，不执行安装命令。|
| -t | 下载测试所需的依赖包。|
| -u | 更新包，包括其依赖包。|
| -v | 查看并执行命令。|

go test

该命令go的是命令，包含单元测试和性能测试

  • 测试源码文件是名称以“_test.go”为后缀的
  • 测试函数一般是以“Test”为名称前缀
  • 测试函数有一个类型为“testing.T”的参数声明的函数.
    | 参数 | 说明 |
    |—–|—–|
    | -c | 生成用于运行测试的可执行文件，但不执行它。|
    | -v | 显示所有测试函数执执行细节。 |
    | -i | 安装/重新安装运行测试所需的依赖包，但不编译和运行测试代码。|
    | -o | 指定用于运行测试的可执行文件的名称。|
    | -timeout t | 单个测试超时时间。|

    go install

    和 go build 参数相同，将生成文件拷贝到 bin、pkg 目录。优先使 GOBIN 环境变量
    所指定目录。

    go vet

    是一个用于检查Go语言源码中静态错误的简单工具
